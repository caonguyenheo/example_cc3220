110 /*
111  * The bits in the pin mode macros are as follows:
112  * The lower 8 bits of the macro refer to the pin, offset by 1, to match
113  * driverlib pin defines. For example, UARTCC32XX_PIN_01_UART1_TX & 0xff = 0,
114  * which equals PIN_01 in driverlib pin.h. By matching the PIN_xx defines in
115  * driverlib pin.h, we can pass the pin directly to the driverlib functions.
116  * The upper 8 bits of the macro correspond to the pin mux confg mode
117  * value for the pin to operate in the UART mode. For example, pin 1 is
118  * configured with mode 7 to operate as UART1 TX.
119  */
